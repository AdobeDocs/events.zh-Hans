# Adobe Journey Optimizer中的AEM内容片段：新增功能及其使用方式

在本次Experience League Live会话中，我们将介绍Adobe Journey Optimizer (AJO)中Adobe Experience Manager (AEM)内容片段的最新增强功能，以使您的内容团队和历程参与者能够更高效地合作，减少摩擦并更好地控制。

[![ExL LIVE 2026年7月16日](assets/WebBannerExlLive-July16-2026.png)](https://youtube.com/live/lrdorB0LgFo){target="_blank"}

无论您是管理内容库的AEM作者，还是构建个性化历程的AJO从业者，此课程都涵盖新增功能以及如何使其正常工作。

加入我们，详细了解如何：

从多个AEM配置（包括创作层、发布层和经过身份验证的发布层）获取内容片段，并在您的历程和营销活动中切换使用的源存储库。

从AJO中新的专用内容片段列表页面管理片段生命周期，您可以在此页面识别不同步的片段，并触发手动同步以保持最新。

选择片段后，可在整个消息中保留内容片段上下文，以便作者无需重新选择即可跨内容块重用片段字段。

使用区域设置和变体支持以更刻意地使用相同内容片段的替代版本。

如果您在AEM上，请直接在AJO中访问Managed Services内容片段。 在配置设置中添加一次AEM Managed Services存储库URL，您便已准备就绪。


您将清楚地了解AEM和AJO如何在生产内容工作流中组合，并亲手实践insight以了解新功能，这些功能使片段更可用、更易于管理，并且更易于大规模生产。

非常适合就内容进行协作并想要减少团队间切换冲突的AEM作者和AJO从业者。
