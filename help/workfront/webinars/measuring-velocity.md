# 咨询专家 — 测量速度

了解如何使用测量和跟踪速度 [!DNL Workfront] 报告。 该研讨会于2019年8月14日录制。

>[!VIDEO](https://video.tv.adobe.com/v/341057/?quality=12)

## 演示文稿中使用的自定义字段

通过复制和粘贴以下计算来节省一些时间。

**首次提交日期**

格式：日期

计算：

```
IF(ISBLANK(First Commit Date),Default Baseline.Planned Completion Date,First Commit Date)
```

**首次持续时间**

格式：文本

计算：

```
IF(ISBLANK(First Duration),Default Baseline.Duration,First Duration)
```

**工作与提交比率**

Format：Number

计算：

```
ROUND(DIV(Actual Duration,First Duration),1)
```

**工作与提交比率状态**

格式：文本

计算：

```
IF({Work-to-Commit Ratio}>2,"Terrible",IF({Work-to-CommitRatio}>1.6,"Poor",IF({Work-to-Commit Ratio}>1.2,"Not Bad","Exc ellent")))
```

**调整后的速度**

Format：Number

计算：

```
ROUND(DIV(Actual Duration,Duration),1)
```

**已调整速度状态**

格式：文本

计算：

```
IF(Adjusted Velocity>2,"Terrible",IF(Adjusted Velocity>1.6,"Poor",IF(Adjusted Velocity>1.2,"Not Bad","Excellent")))
```

## Q&amp;A

**问题**

您好，感谢您组织本次网络研讨会。 我有一个关于Workfront的菲尔德的问题。 在我们的系统中，我们创建了一个名为“状态”的自定义字段，它是状态和条件的组合。 该状态域包含上千个项目中的许多雕像，这对于我们的Tableau数据提取非常重要。 但是，现在我们要消除此字段并使用条件字段，改为使用本机字段。 您知道如何在不丢失数据的情况下翻转此字段吗？ 我现在唯一能想到的是在不丢失数据的情况下手动将数据从项目切换到项目。

**回答**

对于此类情况，您可以使用筛选和批量编辑来根据状态自定义字段半自动填充条件字段。

步骤如下：

1. 确定要映射到条件值的状态值。 例如，假设您的状态值为“Late”和“Very Late”，并且这两个值都映射到“In Trouble”的条件值
1. 创建一个项目报告，其中显示所有状态值为“延迟”和“非常延迟”的项目
1. 运行报表。 确保您显示所有项目（请参阅报表右下方的选项）
1. 单击列标题栏中报表左上角的复选框。 这将选择报告中的所有项目
1. 单击报告列表上方的编辑按钮
1. 将条件类型设置为手动
1. 将“条件”字段设置为“存在问题”
1. 单击保存更改


**问题**

如何定义“优秀”、“不差”等？

**回答**

这只是个例子，但这是我的设置。 首先计算两个索引：

调整后的速度

此公式为“实际持续时间/计划持续时间”（存储在项目的“持续时间”字段中）。 由于每次重新计划项目时项目的计划持续时间都会更改，因此计划持续时间表示最终重新计划。

工作与提交比率

此公式与“调整后周转率”公式相似，不同之处在于，我们希望使用首先向客户承诺的计划持续时间，而不是使用最终重新计划中的“计划持续时间”值。 我们假设原始基线包含此信息（并且我们计划从现在起要求我们的项目经理以这种方式计划他们的项目，以便我们能够捕获准确的数据）。 我们从原始基线中捕获了此持续时间值，并将其称为“首次持续时间”。

通过将实际持续时间除以计划持续时间或首次持续时间，我们得到一个数字，该数字可以告诉我们我们距离目标有多近。 如果计划持续时间或第一个持续时间等于实际持续时间，则索引将等于1。 如果实际持续时间大于1，则答案将大于1。 数目越多，我们见面的日子就越差。

因此，考虑到所有这些因素，我决定为调整后的周转率和工作与承诺量比率指定以下状态：

* 1.1或更低版本，我称之为“优秀”。
* 1.2到1.5，我叫“不错”。
* 从1.6到1.9，我叫穷人。
* 2个以上，我叫“很糟”。

**问题**

工作人员需要做哪些工作来跟踪完成项目所需的时间？

**回答**

我们不是跟踪实际花费在项目上的小时数，我们只是跟踪和比较持续时间。 但是，如果您正在跟踪小时数，并且希望使用计划小时数中的实际小时数来计算周转率，则可以通过将计划小时数与实际小时数进行比较来完成此类型的报告。 您还需要从原始基线捕获计划小时数。

**问题**

能否提供用于Velocity的过滤器？

**回答**

我对Velocity报表使用了两个筛选规则：

* 类别>>> ID >>等于>> WPI项目数据（这是包含所有计算字段的自定义表单）。 我只想查看使用此自定义表单的项目)
* 项目>>状态>>等于>>完成(我只希望查看已完成的项目，因为它们有一个实际持续时间值，该值表示完成所有项目所需的时间。 当前项目无法提供准确的实际持续时间来计算速度)

我还添加了其他筛选条件，以使我的报表保持足够小，以便用于网络研讨会，但在您的生产环境中，您可能希望查看特定时间段内具有WPI自定义表单的所有项目。 您可能需要根据项目的实际完成日期进行筛选。

**回答**

否，基线不包含在复制的项目中

**问题**

对于任务审批流程，能否向我展示如何创建一个报告，按项目中的任务为各个审批者提供带有时间/日期戳的审核追踪？

**回答**

创建任务报告。 在“列”（视图）选项卡中，单击“添加列”。 在“Show in this column：（在此列显示：）”框中，键入“approv”。 这将向您显示可报告的各种批准字段。 我建议您先为所有内容（除任何ID外）添加一列，然后可以查看显示的信息。

接下来，转到过滤器选项卡，并为以下项添加过滤器规则：

任务>>为审批>>等于>> True。 这将仅显示已附加批准的任务。

根据需要添加其他过滤器。

**问题**

我想创建一个验证报告。 项目列表，显示其有多少验证以及该验证有多少版本。

**回答**

创建文档报告。

默认视图将显示版本号。 您希望将其保留在那里，但您可以更改任何其他列。

按项目名称对报告进行分组。

按当前版本筛选报告：验证ID不为空。

这将为您提供每个项目中所有校对的列表。 每个验证都有一行并显示版本号（与版本总数相同）。

**问题**

能否在任务级别使用velocity？ 而不是项目级别？

**回答**

是，但您需要复制项目自定义表单，并从中创建任务自定义表单。 然后，您需要编辑“首次提交日期”字段中的计算，并将对“默认基线”的引用更改为“默认基线任务”。 对于“第一持续时间”，执行相同操作。 之后，您可以将任务自定义表单附加到要测量的任何任务。 您需要创建任务报告，而不是这些项目报告。 但是，您仍然需要确保将原始项目基线设置为默认基线。 所有任务数据与项目数据都保留在相同的基线中。

**问题**

是否必须由项目所有者手动设置第一个提交日期？ 还是从现有字段中提取？

**回答**

首次提交日期是从默认基线中捕获的。 只要默认基线是原始基线，就会在项目首次设置为“当前状态”时显示项目的计划完成日期。

**问题**

自定义表单中的计算字段仍需要定期刷新是否正确？ 或者，这会在一夜之间（或在其他某个触发点）自动发生吗？

**回答**

计算字段会重新计算：

* 当用户编辑对象时
* 使用激活的重新计算自定义表达式批量编辑
* 使用选定的“更新以前的计算”选项对表单进行修改

**问题**

如果velocity考虑持续时间，那么项目偏好设置中的完成百分比是否应基于持续时间？

**回答**

否，“项目首选项”选项仅引用完成百分比的计算方式。 持续时间值本身不受此设置的影响。

**问题**

首次持续时间与计划持续时间之间有何区别？

**回答**

第一个持续时间是您最初向客户承诺的项目将花费的天数。 我们从项目从“计划”更改为“当前”时记录的原始基线中获取此数字。

计划持续时间是从项目开始日期到计划完成日期的天数。 最初，这两个持续时间相同，但如果重新计划了项目并且计划完成日期更改了，则计划持续时间也会更改。

Velocity报表的值来自我们能够查看与第一个持续时间相比，计划持续时间发生了什么变化。 我们可以追溯到项目从“计划”更改为“当前”时，我们最初开始记录基线时。

**问题**

是否可以按颜色设置工作程序，以使它们在所有报表中均相同？

**回答**

如果您在任务报表中按“分配至”>“姓名”分组，则可以在“图表”标签中为特定工作人员分配颜色。 只需在“图表”选项卡中选择指定给>>名称框旁的“自定义颜色”选项，并为每个工作进程添加一种颜色。

**问题**

是否有创建报告以在文档库中查找特定命名文档的方法？ 要测量特定文档是否存在的仪表板的一部分。

**回答**

可以。您需要创建文档报告。 您每次运行报告时，似乎都需要提供特定的文档名称。 如果是这种情况，我建议转到报告选项并选择报告提示。 为Document >> Name添加提示。

**问题**

您能否选择未在图表选项卡中列出的颜色/十六进制值，但它是新颜色，是新十六进制值，例如我的品牌颜色中的新颜色，以便允许我自定义报表？

**回答**

是，您可以输入可能能够找到的任何RGB代码。 这是一个标准代码，用于表示颜色中包含的红色、绿色和蓝色数量。 Workfront将接受从000000到FFFFFF的任何十六进制值，因此如果您知道品牌颜色的代码，则可以使用该代码。

**问题**

能否在仪表板中重新声明报告的定义（提供每个报告衡量标准的说明）？

**回答**

调整后的速度状态图表

> 这显示了我们在比较项目的实际持续时间与计划持续时间时在按时完成项目方面做得如何。 计划持续时间已调整，因为项目在其生命周期内进行了重新规划。

工作与提交率状态图表

**问题**

如何将新数据添加到x轴？

**回答**

当您在报告中按任何内容分组时，将允许您创建图表。 然后，可以在“图表”选项卡中设置X轴或Y轴。

**问题**

能否跨越首次持续时间与实际持续时间之间的差异？

**回答**

第一个持续时间是您最初向客户承诺的项目将花费的天数。 我们从项目从“计划”更改为“当前”时记录的原始基线中获取此数字。

实际持续时间是从项目开始日期到实际完成日期的天数。

**问题**

项目基线如何影响此报告？

**回答**

项目的原始基线包含计划完成日期和项目首次更改为当前状态时存在的计划持续时间。 如果您的流程是在将项目设置为当前之前对其进行规划，则这将表示您承诺在哪天之前完成项目。

**问题**

是否有办法将新表单批量应用于旧项目？

**回答**

可以，您可以从列表中选择多个项目。 执行此操作后，列表左上角将显示“编辑”选项。 选择多个对象时单击“编辑”会将您置于我们所说的“批量编辑”中。 您可以通过这种方式将自定义表单添加到多个项目。

向大量项目添加自定义表单的快捷方式是创建一个报表，并在筛选后仅包含所需的项目。 然后，只需单击列表中第一个复选框上方的复选框，即可选择整个列表，而不是单独选择项目。

**问题**

是否可以从分配报表的分组内删除重复条目，但不能跨分组删除？

**回答**

考虑列表报表中的分组的最佳方法是：

首先，使用“筛选器”选项卡可控制要在列表中显示的项目。 将不会有重复条目。 滤镜将应用于每个对象。 如果它通过过滤器，则它将在列表中显示一次，否则不会显示。

下一个分组将应用于筛选列表。 分组可标识列表中对象的一件事，如它所包含的项目组合的名称（您不能对事物列表进行分组，而只能对单个事物进行分组）。 然后，具有相同值的所有对象将显示在该分组中，就像同一项目组合中的所有项目一样。 任何未选择项目组合的项目将显示在名为“无值”的分组中。

因此，任何对象都不可能出现在多个分组中。 而对象是否出现在列表中完全由过滤器控制（以及运行报表的人员是否有权查看该对象）。

**问题**

是否建议使用任何其他报表来跟踪Velocity？ 只推荐一个高级别的页面就很棒，因为我知道没有足够的时间来详细介绍。

**回答**

与任何报告一样，您首先需要做的就是决定要了解的内容。 下一步是访问该信息，在某些情况下，这意味着您需要开始跟踪该信息。

我决定比较实际持续时间与两种计划持续时间的一个原因是，我认为它提供了有关velocity的有趣见解。 数据也已经可用，因此我不必开始跟踪。 通过一些计算，我可以用我可以报告的表格提取数据。

但您还不妨决定跟踪有关要报告的任务或项目的其他信息。

Workfront没有任何内置velocity报表，因此我建议您和您的团队就确定速度所需的知识进行头脑风暴，然后查看您需要跟踪的内容。

**问题**

能否在COLUMN级别计算任何内容？ 而不是从自定义表单中调用计算字段？

**回答**

在文本模式下可以使用值表达式来执行这些计算。 但是，我们不可能执行首次持续时间或首次提交日期，我们需要捕获那些不会更改的地方。

至于工作与提交比率状态和已调整速度状态，这些字段需要自定义，以便我们可在“图表”选项卡中使用它们。 “图表”选项卡无法识别文本模式分组，它们必须是自定义字段。 由于我们需要工作与提交的比率和调整后的速度来计算这些状态，因此也需要它们作为自定义字段。 所以在这种情况下，它们都需要是自定义字段，但最好考虑这两种方式，并选择效果最好的方式。 谢谢你提问。

**问题**

我们的项目经常因客户延迟或整个过程中的更改而更改。 我们的报告可能全部显示为“糟糕”。 跟踪更改原因的建议是什么？ 我们考虑在文档级别添加一个自定义表单，用于报告更改的原因（内部更改或客户更改），但想知道最佳实践是什么。

**回答**

最佳实践是使用下拉菜单来跟踪此情况。 在开始使用时，请根据您能想到的所有原因添加“原因”，然后添加“其他”选项以捕获不在列表上的原因。 如果新原因看起来或变得很常见，请将其添加到您的下拉菜单中。 您可以轻松地在下拉列表中报告内容，并且可以分组此字段（您无法分组复选框或多选下拉列表）。

只是对此的又一个评论。 您可能不希望在Velocity报表中包含所有项目。 如果你在修复错误，或者去“以前没有人去过的地方”，你可能不会像在建一个你以前已经建过很多次的房子一样，在竣工日期前做出承诺。

因此，请确保将Velocity报表的重点放在它可以帮助您实现目标的位置。

**问题**

如果我将某个项目的默认基线设置为“原始”，然后将该项目从当前基线中移除并放回当前基线，它是否会更改我的默认基线？

**回答**

可以。每次将状态更改为当前时，您都会获得新的基线，并且它将是新的默认值。 但所有先前的基线仍将存在，您可以手动将原始基线再次设置为默认基线。

**问题**

是否可以在报告中设置哪些字段可编辑？ 我可以为某些字段设置限制吗？

**回答**

您可以限制对自定义表单中字段的查看和编辑权限。 您需要将字段包含在部分中，并在部分设置中，选择用户查看或编辑部分中的字段所需的权限。

**问题**

能否创建在文档库中查找特定命名文档的报告？

**回答**

可以。您需要创建文档报告。 您每次运行报告时，似乎都需要提供特定的文档名称。 如果是这种情况，我建议转到报告选项并选择报告提示。 为Document >> Name添加提示。

**问题**

在报表中，为什么值可用作列，但不可用于选择或分组。 例如：问题源。

**回答**

列可能是可查看的，但不可用于分组，主要原因是它可能包含列表，如复选框自定义字段或任务分配。 不允许对列表进行分组。

**问题**

如何在报告中将小时数输入的时间以及实际执行小时数分隔开（按哪些字段划分）？

**回答**

Hour对象的Entry Date字段是指小时数的日期。 这使得输入日期不同于其他对象上的输入日期，其他对象上的输入日期是指创建对象的日期。 即使没有小时的创建日期，也会有“上次更新日期”，最初是创建日期，然后是此后编辑该小时的任何日期。

**问题**

从报表的角度来看，我们如何才能访问基线数据？ 我有一个项目包含多个基线。 我想了解各个任务在每个基线中的计划情况。 是否有办法编写一个显示每个基线的项目计划的报告？

**回答**

报表将显示要查看的当前设置为默认基线的任何字段，因此您可以更改基线并刷新报表以查看带有新基线的字段。

但是，如果您想以图形方式查看有关任务的信息，可以使用甘特图功能执行此操作。 在任务列表中打开甘特图（使用“自动保存”旁边右上角的甘特图），然后转到正下方和右侧的“设置”图标，并单击它。 选中“基线”框，它将显示所有基线。 您可以一次选择一个，并在“甘特图”视图中查看任务的更改。

**问题**

如何创建报告，以查找在定义的时间段（例如上个月）其状态的更改。

**回答**

Workfront中的Analytics功能提供了一种查看历史数据（包括状态更改）的快捷方式。

但您也可以使用备注报告获取状态更改信息。 如果您正在跟踪项目状态字段，则可以进行筛选以查看项目的状态更改。

因此，请首先转到“设置”>“界面”>“更新馈送”，并确保项目状态是被跟踪的内置字段之一。 如果不是的话，你需要添加它。

现在，创建“注释”报告并执行以下操作：

在列（视图）选项卡中：

* 将“注释文本”列替换为“审核文本”。 这将显示有关状态从更改为什么的信息
* 保留“项目：名称”和“输入日期”列
* 单击“输入日期”列，然后在“列设置”面板中选中“按此列排序”。 如果要查看顶部的最近状态更改，请按降序排序。

在“分组”选项卡中：

* 按项目分组：名称

在“过滤器”选项卡中，创建以下过滤器规则：

* 注>>审核类型>>等于>>状态更改
* 添加任何其他规则以按注释输入日期进行筛选。 您可能希望将此项排除在过滤器之外，改为使用报表提示
* 根据需要对项目、项目组合或其他数据进行筛选。

**问题**

作为Planner ，您是否可以提取其他用户的报告？

**回答**

Planner可以创建报告并与任何用户共享，即使与不是用户的人共享。 查看报表时，请转到“报表操作”>“共享”，然后单击“报表访问”框右上角的齿轮图标。 选择“将此设为向外部用户公开”。 选项。 您可以复制提供的链接并将其发送给任何人。 他们将在浏览器中实时查看报告。
